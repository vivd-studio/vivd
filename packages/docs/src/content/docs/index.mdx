## Recommended first read

If you are new to Vivd, start with [Getting Started](/getting-started/).

From there, the usual path is:

1. choose the right starting point: [Create from Scratch](/create-from-scratch/) or [Import an Existing Website](/import-existing-website/)
2. refine the project in [Edit in Studio](/edit-in-studio/)
3. finish with [Publish Your Site](/publish-your-site/)

If you are self-hosting, keep [Self-Hosting](/self-hosting/) and [Instance Settings](/instance-settings/) close by from the start.

<Aside type="note" title="What this docs site covers">

- Product workflows for creating, editing, publishing, and operating sites in Vivd.
- The runtime model behind Vivd's isolated Studio machines and real project workspaces.
- The solo self-host install path and first-run setup.
- Instance-level self-host admin settings and defaults.
- Domain registration, verification, and publish-target behavior.
- Transactional email identity and delivery-health operations.
- Plugin setup for Contact Form and Analytics.
- Team-access guidance for agencies and customer-facing projects.
- Practical troubleshooting for common issues.

This docs site intentionally does **not** cover internal architecture, Fly machine internals, contributor workflows, or deep deployment-ops runbooks.
</Aside>
