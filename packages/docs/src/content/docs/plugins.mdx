Plugins let a Vivd project do more than render pages. In the current product surface, the main plugin workflows are Contact Form and Analytics.

On self-hosted installs, plugin availability can also be controlled instance-wide before
you do any project-level setup. Use [Instance Settings](/instance-settings/) and the
adjacent `Plugins` admin surface when a plugin appears unavailable across the whole
install rather than just one project.

<Aside type="tip" title="When to do plugin setup">
  Configure plugins before the final publish pass, not after launch day. That keeps form delivery, analytics expectations, and handoff steps inside the same review cycle.
</Aside>

## Recommended launch workflow

1. Finish the main site structure in Studio.
2. Configure the plugins that matter for launch.
3. Test the form flow and analytics expectations.
4. Run through the publish checklist.

## Re-check plugins when

- The customer-side reviewer changes and recipients or owners need updating.
- The project domain changes and you want a fresh launch review.
- The site gains new lead flows or the traffic expectations change after launch.
